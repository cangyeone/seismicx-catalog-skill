# Association And Location

Use this reference when choosing REAL, GaMMA, the built-in grid locator, or `cangyeone/bayes_location`.

## Choosing The Associator

Use GaMMA when:

- Picks are dense, noisy, or generated by deep learning models.
- The user wants probabilistic clustering and can install Python dependencies.
- Uniform or 1D velocity assumptions are acceptable for association.

Use REAL when:

- The user wants a proven rapid association/location workflow with explicit travel-time windows.
- A local REAL binary can be compiled.
- The user can provide regional `R`, `G`, `V`, and `S` parameters and a suitable station/pick conversion.

Use the helper's `simple` method only as a smoke test for tiny examples. Do not present it as a scientific association method.

## Building REAL

The helper can clone and build REAL into a local tools directory:

```bash
python scripts/seismicx_catalog.py build-tools \
  --tool real \
  --tools-dir external \
  -o work/build_manifest.json
```

If the automatic Makefile search fails, inspect `external/REAL`, read its README/user guide, install `gfortran`, and run the repository's documented build command manually. Record the executable path in `work/build_manifest.json` or final notes.

REAL command execution is project-specific because different deployments convert picks into REAL input files differently. The helper prepares a workspace and can run a template:

```bash
python scripts/seismicx_catalog.py associate \
  --method real \
  -p work/picks.csv \
  -s stations.csv \
  -o work/events_real.csv \
  --workdir work/real \
  --real-command "<REAL command using {workdir} {stations} {picks} {output} {R} {G} {V} {S}>"
```

## Running GaMMA

Install GaMMA and run:

```bash
python scripts/seismicx_catalog.py associate \
  --method gamma \
  -p work/picks.csv \
  -s stations.csv \
  -o work/events_gamma.csv \
  --assignments work/assignments.csv \
  --vp 6.0 \
  --vs 3.5 \
  --min-picks-per-eq 5 \
  --min-p-picks-per-eq 3
```

Tune `dbscan_eps`, `oversampling_factor`, `min_picks_per_eq`, `max_sigma11`, and velocity values for the local network. For regional phases, confirm that the selected `Pg/Sg` or `Pn/Sn` labels are mapped to P/S classes expected by the associator.

## Built-In Grid Location

The grid locator is a baseline, not a replacement for a production locator. Use it to create a transparent first catalog or to validate association output:

```bash
python scripts/seismicx_catalog.py locate \
  --method grid \
  -p work/picks_associated.csv \
  -s stations.csv \
  -v velocity_model.csv \
  -o work/events_located.csv
```

Requirements:

- Picks must include `event_id`.
- Station coordinates must be valid.
- Velocity model must be supplied for production results.

QC the resulting RMS, depth distribution, station coverage, and map before calculating ML.

## bayes_location Integration

Use `cangyeone/bayes_location` when the user explicitly wants Bayesian location or the local project already has an adapter.

Export a JSON payload if no adapter command exists yet:

```bash
python scripts/seismicx_catalog.py locate \
  --method bayes \
  -p work/picks_associated.csv \
  -s stations.csv \
  -v velocity_model.csv \
  -o work/bayes_location_input.json
```

Run a local adapter command when available:

```bash
python scripts/seismicx_catalog.py locate \
  --method bayes \
  -p work/picks_associated.csv \
  -s stations.csv \
  -v velocity_model.csv \
  -o work/events_bayes.csv \
  --bayes-repo external/bayes_location \
  --bayes-command "python run_location.py --picks {picks} --stations {stations} --velocity {velocity_model} --output {output}"
```

Always record the velocity model, priors, uncertainty assumptions, and `bayes_location` commit.
