# Data Contracts

Use these contracts when normalizing inputs and exchanging data between pickers, associators, locators, magnitude code, and focal-mechanism code.

## Waveforms

- Accept MSEED, miniSEED, SAC, SEED, SEG-Y, GSE, GCF, and other formats supported by `obspy.read`.
- Preserve the original file path in all downstream pick rows as `waveform_path`.
- Prefer one trace id format everywhere: `NET.STA.LOC.CHA`. Empty location codes may be represented as an empty field in CSV and omitted only in display text.
- Never assume a directory naming convention. Always scan headers first with:
  `python scripts/seismicx_catalog.py scan -w <waveforms> -o work/waveforms.csv`

## Station Metadata

Canonical station CSV columns:

```csv
station_id,network,station,location,longitude,latitude,elevation_m
YN.C2714.40,YN,C2714,40,105.1287,27.6082,1612.0
```

Legacy whitespace station files are also accepted:

```text
YN C2714 40 105.1287 27.6082 1612.0
```

For ML, station response is required for calibrated amplitudes. Use StationXML where possible. If only RESP/dataless files exist, convert them before the ML step or explicitly mark ML as uncalibrated/raw-scaled.

## Velocity Model

Use a simple CSV or whitespace model for baseline location and for configuring external engines:

```csv
depth_km,vp_km_s,vs_km_s
0,5.8,3.35
10,6.2,3.58
20,6.8,3.92
```

Always confirm whether depths are positive downward and velocities are km/s. For `bayes_location`, pass the model in the format required by the local adapter and record the adapter command.

## Picks

Canonical pick CSV columns:

```csv
pick_id,event_id,waveform_path,trace_id,network,station,location,channel,phase,time,score,snr,amplitude,polarity,polarity_quality,polarity_score,picker
p00000001,,/data/day001/YN.C2714.40.EIZ.mseed,YN.C2714.40.EIZ,YN,C2714,40,EIZ,Pg,2023-05-03T04:08:29.140000Z,0.58,2.23,80.1,U,E,2.4,classic-sta-lta
```

Rules:

- `time` must be UTC ISO text.
- `phase` may be generic (`P`, `S`) or regional (`Pg`, `Sg`, `Pn`, `Sn`). Keep the user's selected phase labels when possible.
- `event_id` is empty before association and filled after association.
- `polarity` is `U`, `D`, or `N`; `polarity_quality` is `I`, `E`, or empty.

## Events

Canonical event CSV columns:

```csv
event_id,origin_time,latitude,longitude,depth_km,magnitude,magnitude_type,rms,n_picks,method
ev000001,2023-05-03T04:08:19.120000Z,28.11,105.14,10.0,4.1,ML,0.21,18,grid-least-squares
```

`events.csv` may be generated by GaMMA, REAL, the grid locator, or `bayes_location`. If an external tool emits a different schema, convert it to the canonical columns before ML, analysis, plotting, or focal-mechanism calculation.
